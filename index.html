Introduction

React is a JavaScript library for building user interfaces. React has been designed from the start for gradual adoption, and you can use as little or as much React as you need. Whether you want to get a taste of React, add some interactivity to a simple HTML page, or start a complex React-powered app, the links in this section will help you get started.

Hello World

The smallest React example looks like this:

ReactDOM.render( <h1>Hello, world! </h1>, document.getElementById('root'));
Introducing JSX

Consider this variable declaration:

const element = <h1>Hello, world!</h1> ;

It is called JSX, and it is a syntax extension to JavaScript.

Rendering Elements

Elements are the smallest building blocks of React apps.

const element = <h1>Hello, world!</h1> ;
ReactDOM.render(element, document.getElementById('root'));
Components and Props

Components let you split the UI into independent, reusable pieces, and think about each piece in isolation. This page provides an introduction to the idea of components.

function Welcome(props) { return <h1>Hello, props.name </h1>;}
State and Lifecycle

the state, in React, is where we keep our component data.

constructor(props) { super(props); this.state = {date: new Date()}; }

The React component, like anything else in the world, goes through the following phases.

  • Startup
  • Assembly
  • Update
  • Disassembly
Handling Events

Handling events in React elements is very similar to handling events in DOM elements. There are some syntax differences:

  • React events are named using camelCase instead of lowercase letters.
  • With JSX you pass a function as an event handler instead of text.
